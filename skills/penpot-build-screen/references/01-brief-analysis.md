# 01 — Brief analysis

Turn a loose request into a buildable plan before touching the canvas.

## Extract
- **User goal** — the one thing this screen must let the user do.
- **Content inventory** — the real sections/blocks and their data (nav, hero, list, form, footer…).
- **Platform & viewport** — desktop 1440, tablet, mobile 375; pick the primary.
- **Tone/brand** — feeds the style profile (`02-style-profiles.md`).

## Produce
- An ordered **section list** (top → bottom) — each becomes one build step.
- A note of which **existing components** map to which section.
- Open questions / assumptions to confirm at the Phase 0 checkpoint.

A vague brief ("make it nice") is converted into Context / Objective / Inputs / Constraints /
Acceptance Criteria (see SKILL.md §5) before building.
